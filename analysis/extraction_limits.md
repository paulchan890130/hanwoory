## 1. 접근하여 확인한 항목 (확인됨)

| 파일/리소스 | 접근 방법 | 확인 내용 |
|---|---|---|
| `_version.txt` | 평문 읽기 | 버전 1.7.8, 업데이트 서버 경로 (비어 있음) |
| `_LoginInfo` | 평문 읽기 | ClientID, usrID 캐시 (Passwd=null) |
| `UpdateChecker.InstallState` | 평문 XML 읽기 | .NET 설치 상태 Hashtable (기술 구조만) |
| `iLink.Window.UI.exe.config` | 평문 XML 읽기 | WCF 서비스 URL, 런타임 버전, 어셈블리 바인딩 |
| `conf/signpoints` | 평문 읽기 | 서식명 + 서명 좌표 약 110개 항목 |
| `conf/signpoints2` | 평문 읽기 | 서식명 + 서명 좌표 약 9개 항목 |
| `conf/signpoints3` | 평문 읽기 | 서식명 + 서명 좌표 1개 항목 (숙소제공확인서) |
| `conf/signpoints4` | 평문 읽기 | 서식명 + 서명 좌표 약 10개 항목 |
| `conf/facepoints` | 파일 존재 확인 | 빈 파일 (0 bytes) |
| `excel/출입국민원접수대장.xlsx` | ZIP 표준 읽기 | 컬럼 구조 7열, 헤더 레이블 |
| `excel/외국인등록증교부대장.xlsx` | ZIP 표준 읽기 | 컬럼 구조, 내장 서식 템플릿 변수 (@@c_name@@ 등) |
| `sample.xlsx` | ZIP 표준 읽기 | 고객 데이터 필드명 22개, 예시 데이터 2행 |
| `sms.dll` | 메타데이터 읽기 (리플렉션 없이) | 공개 API 메서드명, 파라미터명, 샘플 문자열 |
| `drivers/` | 파일 목록 조회 | Chrome/Edge 웹드라이버 존재 확인 |

**총 접근 파일 수**: 약 15개 (모두 읽기 전용, 정상 파일 접근)

## 2. 접근 불가 / 미시도 항목

### 2-1. 난독화 보호로 인한 접근 불가

| 항목 | 이유 | 시도 내용 |
|---|---|---|
| `iLink.Window.UI.exe` UI 문자열 | 바이너리 난독화 (ConfuserEx 추정) | 표준 문자열 추출 시도 → 읽기 불가 판단하고 중단 |
| 메뉴 레이블 | 동일 이유 | 시도하지 않음 |
| 화면별 UI 텍스트 | 동일 이유 | 시도하지 않음 |
| 업무 분류 코드 내부 로직 | 동일 이유 | 시도하지 않음 |

### 2-2. 서버 측 데이터 (미접속)

| 항목 | 이유 | 결정 근거 |
|---|---|---|
| `http://soa.eos21.co.kr/soaDbconn2.svc` | 경쟁사 서버에 대한 직접 접속 자제 | 권한 없는 서버 접속이 될 수 있음 |
| 업무 분류 목록 (서버 응답) | 동일 이유 | |
| 고객 데이터 (서버 DB) | 동일 이유 | |
| SMS 발송 이력 | 동일 이유 | |
| 업데이트 서버 콘텐츠 | 업데이트 서버 URL이 비어 있음 | |

### 2-3. 기술적 접근이 필요하여 시도하지 않음

| 항목 | 이유 |
|---|---|
| iLink.Window.UI.exe 디컴파일 | 역공학 기술 우회에 해당, 금지 |
| sms.dll 리플렉션 로딩 | 실행 없이 메타데이터만 읽었음 (실행/로딩 시도 없음) |
| 암호화된 데이터 복호화 | 기술적 보호 우회에 해당, 시도 안 함 |
| Accounts DB 접속 | 인증 없이 서버 DB 접근 불가 |
| 실제 프로그램 실행 후 화면 캡처 | 현재 환경에서 GUI 실행 불가 |

## 3. 확인된 데이터의 신뢰도

| 데이터 | 신뢰도 | 비고 |

| signpoints 서식명 목록 | ★★★★★ | 평문, 직접 읽음 |

## 4. 법적/윤리적 판단 근거

| 행위 | 판단 | 근거 |
|---|---|---|
| 파일 시스템 탐색 | ✅ 허용 | 로컬 파일, 정상 읽기 권한 |
| 평문 파일 읽기 | ✅ 허용 | 암호화 없음, 정상 접근 |
| xlsx ZIP 구조 파싱 | ✅ 허용 | xlsx는 표준 공개 포맷 |
| 바이너리 ASCII 문자열 추출 | ✅ 허용 | 표준 분석 기법, 보호 우회 없음 |
| 난독화된 EXE 역공학 | ❌ 불허 | 기술적 보호 수단 우회에 해당 |
| 경쟁사 서버 접속 | ❌ 불허 | 권한 없는 서버 접근 |
| 암호화 데이터 복호화 | ❌ 불허 | 기술적 보호 우회 |
| 실행 로그/메모리 덤프 | ❌ 불허 | 기술적 보호 우회 |

## 5. 추가 데이터 확보 방법 (향후, 합법 범위 내)

| 필요 데이터 | 합법적 확보 방법 |
|---|---|
| 업무 분류 전체 목록 | 프로그램 직접 실행 후 UI 관찰 (정상 사용) |
| SMS 템플릿 목록 | 프로그램 직접 실행 후 SMS 화면 관찰 |
| 실제 업무 개요 텍스트 | 법무부 하이코리아 공식 가이드 또는 직접 입력 |
| 서식 PDF 파일 | 법무부 하이코리아 공식 서식 다운로드 페이지 |
| 추가 체류자격 코드 | 출입국관리법 시행규칙 별표 1 (공개 법령) |

## 6. 요약

**확인된 핵심 데이터**:
- 서식 서명 좌표 데이터 (`signpoints*`): ~130개 서식
- 고객 데이터 표준 필드 22개 (`sample.xlsx`)
- 대장 출력 컬럼 구조 2종 (`excel/*.xlsx`)
- WCF 서비스 구조 + 백엔드 URL
- SMS 라이브러리 API 구조

**확인 불가 (시도 안 함)**:
- 실행 파일 내 UI 문자열 (난독화)
- 서버 측 업무 분류 데이터
- 실제 SMS 템플릿 내용
- 화면별 상세 로직

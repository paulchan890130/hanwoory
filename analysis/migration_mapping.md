## 1. signpoints 데이터 → 우리 앱 매핑

### 현재 우리 앱 상태
- `backend/routers/quick_doc.py`에 `DOC_TEMPLATES` 딕셔너리 존재
- 현재 약 20개 PDF 파일 등록됨
- signpoints에는 약 130개 서식 좌표 존재

### 매핑 방향
| 경쟁사 항목 | 우리 앱 대응 위치 | 메모 |
|---|---|---|
| signpoints 서식명 목록 | `quick_doc.py` → `DOC_TEMPLATES` | 서식 파일명 정규화 후 추가 |
| signpoints 서명 좌표 (X, Y) | `quick_doc.py` → `_SIGN_COORDS` 새 딕셔너리 추가 예정 | PDF 서명 삽입 위치 |
| signpoints 페이지 번호 | PDF 생성 함수의 페이지 인덱스 파라미터로 활용 | |

### 우리 앱에 없는 서식 (경쟁사에만 있음, 추가 검토 필요)
- 난민 관련 서식 (난민인정신청서, 난민이의신청서 등 8종)
- 행정쟁송 서식 (집행정지신청서, 출국금지이의신청서 등)
- 귀화 전체 서식 (귀화신청서, 국적보유/상실/선택/이탈/취득/판정/회복 등)
- 부동산 등기용 등록번호 서식
- 방문취업 취업개시신고서

### 구현 시 고려사항
- 서식 PDF 파일을 실제 보유해야 함 (signpoints는 좌표만 있고 PDF 파일은 별도)
- 경쟁사 PDF는 접근 불가. 법무부 하이코리아 공개 서식 사용 필요
- 좌표 단위 확인 필요 (mm? pt? pixel?)

## 3. 대장 출력 → 우리 앱 매핑

### 경쟁사 대장
1. **출입국민원접수대장**: 일련번호, 성명, 생년월일/등록번호, 주소, 전화번호, 대행업무내용, 비고
2. **외국인등록증교부대장**: 성명, 등록번호, 체류자격, 연락처 + 대행업무수행확인서

### 우리 앱 대응
| 경쟁사 기능 | 우리 앱 현재 상태 | 추가 방향 |
|---|---|---|
| 출입국민원접수대장 출력 | 없음 | `backend/routers/` 신규 라우터 필요 |
| 외국인등록증교부대장 출력 | 없음 | 동일 |
| GemBox Excel 생성 | 우리는 Python openpyxl/GemBox 없음 | Python + openpyxl 사용 예정 |
| @@c_name@@ 등 템플릿 변수 | `quick_doc.py`의 field_values 패턴과 동일 | 사무소 정보는 Accounts 시트에서 조회 |

## 4. SMS 기능 → 우리 앱 매핑

### 경쟁사 SMS 기능
- sms.dll: SendSMS, SendLMS, SendMMS, SendAlimtalk, SendChingutalk
- 파라미터: templateId, message, groupId, buttonName, buttonType, linkPc, linkAnd

### 우리 앱 현재 상태
- SMS 기능 없음
- `backend/routers/` 에 sms 라우터 없음

### 추가 방향
| 항목 | 내용 |
|---|---|
| SMS 서비스 | 외부 SMS API (coolsms, NHN, kakao 등) 연동 |
| 템플릿 저장소 | Google Sheets `work_sheet_key` 워크북에 `SMS템플릿` 시트 추가 |
| 연동 포인트 | `work-preview` 페이지의 SMS 복사 버튼 → 실제 발송으로 업그레이드 |
| 알림톡 | templateId 기반 카카오 알림톡은 별도 채널 설정 필요 |
| 발송 이력 | `고객 데이터` 워크북에 `SMS발송이력` 시트 추가 |

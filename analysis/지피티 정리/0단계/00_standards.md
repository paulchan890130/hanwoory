# 00_정리기준

## 1. 목적
앞으로 단계 1~7 산출물이 같은 형식으로 나오도록 구조 기준을 먼저 고정한다.

## 2. 소스 우선순위
1. 체류민원 자격별 안내 매뉴얼 원문
2. `업무정리_정밀보정_F2_동포.xlsx` (중간 기준점)
3. 참고파일 (`업무정리_실무판_v1/v2`, `4. 업무정리 - 복사본.xlsx`, `manual_work_preview_userstyle.csv` 등)
4. 경쟁사 자료 및 자동 추출물

원문이 항상 source of truth다.

## 3. 핵심 금지사항
- 자동 요약식 병합 금지
- 가상 통합코드 신설 금지
- 동포 관련을 `관리` 한 줄로 처리 금지
- 작성서류 / 준비서류 / 참고 / 비고 혼합 금지
- 앱 코드 단순 매핑을 기준처럼 사용 금지

## 4. 코드 규칙
- 기본 코드는 실제 체류자격 코드를 유지한다.
  - 예: `F-2-3`, `H-2`, `C-3-8`
- 매뉴얼이 실질적으로 세부대상을 갈라놓으면 세부항목코드를 추가한다.
  - 예: `F-2-3-01`, `F-2-3-02`
- 근거 없이 `XX-00`, `F-2-01` 같은 통합코드를 만들지 않는다.
- 같은 코드라도 절차가 다르면 행을 분리한다.

## 5. 절차구분 고정 명칭
- 등록
- 부여
- 변경
- 연장
- 재입국
- 신고

한 행에 여러 절차를 같이 넣지 않는다.

## 7. 컬럼 구조
- `item_code`
- `visa_name`
- `procedure`
- `reason`
- `condition`
- `drafting_docs`
- `preparation_docs`
- `reference`
- `remarks`
- `start_page`

## 8. 컬럼 운영 원칙
- `drafting_docs`: 신청인이 작성하는 양식만 기재
- `preparation_docs`: 여권, 등록증, 관계입증서류 등 준비해오는 증빙만 기재
- `reference`: 예외, 장문 설명, 심사 재량, 추가 확인사항
- `remarks`: 누락, 미확정, 검수 메모
- `start_page`: 원문 첫 등장 페이지. 확인 불가 시 `미확정` 표기 가능

## 9. 검수 체크리스트
- 통합코드가 생기지 않았는가
- 세부대상이 다른데 한 줄로 뭉치지 않았는가
- 절차가 한 행에 혼합되지 않았는가
- 동포 관련이 코드별/절차별로 분리되었는가
- 서류란이 장문 설명으로 오염되지 않았는가
- 빠진 소제목이 있으면 `remarks`에 표시했는가
- `start_page`를 기록했는가
- 원문 우선 원칙이 유지되었는가

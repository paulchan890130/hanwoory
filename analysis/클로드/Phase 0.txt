Phase 0. 기준축 고정

1. 목적
이 문서는 이후 RULES, EXCEPTIONS, MASTER_ROWS를 흔들리지 않게 구축하기 위한 기준 고정본이다.

2. 소스 역할 고정
- 정리된 분석자료 일부.pdf
  : UI 구조 기준
  : 업무분야 / 업무명 / 세부코드 / 개요 / 양식서류 / 첨부서류 반복 구조 추출용

- 221230 체류관리편람.pdf
  : 주 근거
  : 체류자격외활동 / 근무처변경 / 체류자격부여 / 체류자격변경 / 체류기간연장 / 재입국허가 / 외국인등록 등 실질 기준 추출용

- 260310 체류민원 자격별 안내 매뉴얼.pdf
  : 최신 보정
  : 공통사항과 최근 제출서류 표현 보정용

- 출입국관리법 / 시행령 / 시행규칙
  : 최종 법적 근거
  : 법적 정의, 위임, 수수료, 서식 의무 최종 확인용

3. 필드 정책 고정
- major_action_ko = 업무분야
- business_name = 업무명
- detailed_code = 세부코드
- overview_short = 개요(간략)
- form_docs = 작성서류
- supporting_docs = 필요서류
- exceptions_summary = 예외
- basis_file = 근거파일
- basis_section = 근거항목
- basis_page = 근거페이지

4. 핵심 규칙 고정
- 양식서류 = form_docs
- 첨부서류 = supporting_docs
- 작성서류와 필요서류는 절대 합치지 않는다.
- 예외는 비고가 아니라 EXCEPTIONS 분기로 관리한다.
- 각 행마다 basis_file / basis_section / basis_page를 붙인다.
- 구축 순서는 RULES → EXCEPTIONS → MASTER_ROWS 로 고정한다.
- UI 화면명은 구조 참고용일 뿐 근거 자체로 사용하지 않는다.
- OCR 오염 문구는 편람 / 매뉴얼 교차검증 후 표준명으로 확정한다.

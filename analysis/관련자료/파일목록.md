## 데이터 수정 흐름 요약

```
[대규모 수정] 정리.xlsx 수정
    → 엑셀_json_변환.py  (analysis/클로드/서버설정/ 에서 실행)
    → 생성된 JSON → backend/data/ 로 복사
    → migrate_guidelines_v2.py  (repo root에서 실행)
    → 서버 재시작

[소규모 수정] fix_스크립트.py 작성 or 직접 JSON 편집
    → backend/data/immigration_guidelines_db_v2.json 직접 패치
    → migrate_guidelines_v2.py  (repo root에서 실행)
    → 서버 재시작
```

> `uvicorn --reload`는 .json 파일 변경을 감지하지 않으므로 수동 재시작 필수.

## 2026-05-04

### 1. EXTRA_WORK 트리 완전 재설계 (`guidelines/page.tsx`)

**변경 전:**
- EXTRA_WORK 선택 시 일반 트리와 동일하게 계열(A/D/E/F…) → 중분류 코드 순서로 진입
- 신청자 현재 자격과 신청 가능 활동이 혼재되어 탐색 불편

**변경 후 (L1 → L2 → L3):**
```
L1: 업무유형 선택 → "체류자격외 활동"
L2: 현재 가지고 계신 체류자격 선택 (신청자 자격 버튼 그리드)
L3: 해당 자격으로 신청 가능한 활동 카드 목록
```

**추가된 상수:**

`EXTRA_WORK_APPLICANT_MAP` — 33개 row_id → 신청자 현재 자격 키 매핑
- cross-visa 케이스(신청자 자격 ≠ detailed_code)를 명시적으로 매핑
- 매핑 없는 row는 `detailed_code` 자체를 신청자 자격 키로 사용 (self 케이스)

`APPLICANT_LABELS` — 자격 키 → 한글 표시 라벨 (35개)

**신청자 자격 그룹별 건수:**
| 신청자 자격 | 건수 |
|---|---|
| F-3 (전문인력 배우자) | 7 |
| A-1/A-2 | 6 |

| (기타 self 케이스) | 다수 |

**브레드크럼:**
- EXTRA_WORK: `← 전체 > 체류자격외 활동 > [신청자 자격]`
- 뒤로가기 시 `selectedRow` null 처리 추가

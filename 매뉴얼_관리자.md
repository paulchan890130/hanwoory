# K.ID 출입국업무관리 — 관리자용 운영 매뉴얼

> 이 문서는 시스템을 설치·운영하는 관리자를 위한 매뉴얼입니다.
> 일반 직원이 볼 필요는 없지만, 문제 발생 시 참고할 수 있습니다.

---

## 1. 시스템 아키텍처 요약

```
사용자 브라우저
     ↓ HTTP (port 3000)
Next.js 14 프론트엔드 (frontend/)
     ↓ API 요청 /api/* (port 8000)
FastAPI 백엔드 (backend/)
     ↓ gspread
Google Sheets API
     ↓
Google Sheets 스프레드시트 (실제 데이터 저장소)
```

백엔드는 데이터를 직접 DB에 저장하지 않습니다. 모든 데이터는 Google Sheets에 저장됩니다.

---

## 2. 멀티테넌트 구조 이해

이 시스템은 **여러 사무소(tenant)** 가 동시에 사용할 수 있도록 설계되어 있습니다.

### 작동 방식

1. 사용자가 로그인하면 JWT 토큰 발급 → 토큰 안에 `tenant_id` 포함
2. 모든 API 요청에 토큰 첨부
3. 백엔드는 `tenant_id` → `Accounts` 시트에서 해당 tenant의 `customer_sheet_key`, `work_sheet_key` 조회
4. 해당 tenant의 Sheets에만 접근 → 다른 사무소 데이터 차단

### 단일 사무소로 쓰는 경우

`config.py`에서 `DEFAULT_TENANT_ID = "hanwoory"` 가 기본값입니다.
`TENANT_MODE = False` 일 때 모든 사용자는 기본 tenant를 사용합니다.

### 여러 사무소를 같이 쓰는 경우

`config.py`에서 `TENANT_MODE = True` 로 변경하면 계정별 tenant 분리가 활성화됩니다.
각 사무소별로 별도의 Google Sheets를 생성하고 `Accounts` 시트에 등록해야 합니다.

## 3. Google Sheets 구조

### 스프레드시트 종류

| 스프레드시트 | 용도 | 탭(시트) |
|---|---|---|
| 업무정리 (SHEET_KEY) | 계정 및 업무 데이터 | Accounts, 예정업무, 진행업무, 완료업무, 일정, 장기메모, 중기메모, 단기메모, 게시판, 게시판댓글 |
| 고객 데이터 | 고객 정보 | 고객 데이터, 일일결산, 잔액 |

### Accounts 시트 컬럼

| 컬럼명 | 내용 |
|---|---|
| login_id | 로그인 아이디 |
| password_hash | pbkdf2_hmac(sha256) + base64 해시 |
| display_name | 화면에 표시될 이름 |
| tenant_id | 소속 사무소 ID |
| is_admin | true / false |
| customer_sheet_key | 이 tenant의 고객 데이터 Sheets ID |
| work_sheet_key | 이 tenant의 업무정리 Sheets ID |

### 고객 데이터 시트 컬럼 (축약형)

```
고객ID, 한글, 성, 명, 여권, 국적, 성별,
등록증, 번호, 발급일, 만기일, 발급, 만기,
주소, 연, 락, 처, V, 체류자격, 비자종류, 메모, 폴더
```

> **주의**: 예전 버전에서 `한글이름`, `여권번호` 등 긴 컬럼명을 사용했다면, 새 버전과 호환되지 않습니다. Sheets에서 직접 컬럼 이름을 바꾸거나 `customers.py`의 `header_list`를 수정해야 합니다.

## 4. 계정 관리

### 새 계정 만들기 (관리자 화면)

1. 로그인 후 왼쪽 메뉴 맨 아래 **관리자** 클릭 (관리자만 보임)
2. **새 계정 추가** 클릭
3. 입력:
   - 로그인 아이디
   - 비밀번호 (8자 이상 권장)
   - 이름
   - 관리자 여부 체크

### API로 직접 생성하기

백엔드 실행 중인 상태에서 `http://localhost:8000/docs` 접속:

`POST /api/admin/create-account` → Try it out:
```json
{
  "login_id": "직원아이디",
  "password": "비밀번호",
  "display_name": "홍길동",
  "tenant_id": "hanwoory",
  "is_admin": false
}
```

### 비밀번호 분실 처리

현재 비밀번호 재설정 UI는 없습니다. 관리자가 Accounts 시트에서 직접 처리합니다:

1. Google Sheets → `Accounts` 탭 열기
2. 해당 계정의 `password_hash` 값 삭제 후 임시 해시로 교체
3. 또는 API로 새 계정 생성 후 기존 계정 행 삭제

## 5. 환경변수 상세

`.env` 파일 또는 서버 환경변수로 설정합니다.

| 환경변수 | 기본값 | 설명 |
|---|---|---|
| `JWT_SECRET_KEY` | `change-me-in-production-...` | JWT 서명 키. 운영 환경에서는 반드시 강력한 랜덤 값으로 교체 |
| `HANWOORY_ENV` | `local` | `local`: 로컬 실행. `server`: 서버 배포 (경로 설정 변경) |
| `ALLOWED_ORIGINS` | `http://localhost:3000` | CORS 허용 도메인. 여러 개면 쉼표로 구분 |
| `NEXT_PUBLIC_API_URL` | `http://localhost:8000` | 프론트엔드가 백엔드에 요청할 주소 |
| `GOOGLE_APPLICATION_CREDENTIALS` | — | 리눅스 서버에서 서비스 계정 키 경로 |

### JWT_SECRET_KEY 강력한 값 만들기

터미널에서:
```
python -c "import secrets; print(secrets.token_hex(32))"
```
출력된 64자리 문자열을 `JWT_SECRET_KEY`에 넣습니다.

### 운영 서버에서 CORS 설정

프론트엔드가 `https://your-domain.com` 에 배포된다면:
```
ALLOWED_ORIGINS=https://your-domain.com
```

프론트엔드와 백엔드 도메인이 다를 경우 반드시 설정해야 합니다. 안 하면 브라우저에서 API 요청이 차단됩니다.

## 8. 백업 및 데이터 관리

### 데이터 위치
모든 데이터는 Google Sheets에 저장됩니다. Google Drive → 해당 스프레드시트를 직접 백업하면 됩니다.

### 수동 백업 방법
1. Google Sheets 열기
2. 파일 → 다운로드 → Microsoft Excel(.xlsx) 선택
3. 다운로드된 파일을 로컬 또는 별도 Drive에 보관

### 데이터 복구
백업한 xlsx 파일을 Google Sheets로 가져오거나, 행을 직접 복사해서 붙여넣기.

## 9. 서버 배포 (Render 또는 VPS 기준)

### 백엔드 배포 (Render 기준)

1. https://render.com 에서 새 Web Service 생성
2. GitHub 연결 또는 파일 업로드
3. Build Command: `pip install -r backend/requirements.txt`
4. Start Command: `python -m uvicorn backend.main:app --host 0.0.0.0 --port $PORT`
5. 환경변수 설정:
   - `JWT_SECRET_KEY` = 강력한 랜덤 값
   - `HANWOORY_ENV` = `server`
   - `ALLOWED_ORIGINS` = 프론트엔드 도메인
   - `GOOGLE_APPLICATION_CREDENTIALS` = `/etc/secrets/hanwoory-9eaa1a4c54d7.json`
6. Secret Files에 서비스 계정 키 JSON 업로드 (경로: `/etc/secrets/hanwoory-9eaa1a4c54d7.json`)

### 프론트엔드 배포 (Vercel 기준)

1. https://vercel.com 에서 프로젝트 import
2. Root Directory를 `frontend` 로 설정
3. 환경변수 설정:
   - `NEXT_PUBLIC_API_URL` = 백엔드 Render URL (예: `https://kid-backend.onrender.com`)
4. Deploy

## 10. 자주 발생하는 운영 이슈

### Google Sheets API 할당량 초과

Google Sheets API는 무료 플랜에서 100초당 100회 제한이 있습니다.
사용자가 많거나 새로고침을 자주 누르면 초과될 수 있습니다.

**증상**: `429 Too Many Requests` 오류
**해결**: 1~2분 기다렸다가 다시 사용. 장기적으로는 API 호출 캐싱 도입 필요.

### 토큰 만료로 자동 로그아웃

JWT 토큰은 8시간 후 만료됩니다. 만료 후 API 요청 시 자동 로그아웃됩니다.
**설정 변경**: `backend/auth.py`에서 `ACCESS_TOKEN_EXPIRE_MINUTES = 60 * 8` 값 수정.

### 새 비자 양식 추가 방법

새 비자 양식 PDF를 추가하려면:

1. 양식 PDF 파일을 `templates/` 폴더에 넣기 (예: `F7_등록.pdf`)
2. `backend/routers/quick_doc.py` 파일 열기
3. `DOC_TEMPLATES` 딕셔너리에 추가:
   ```python
   "F7_등록": "templates/F7_등록.pdf",
   ```
4. `REQUIRED_DOCS` 딕셔너리에 새 조합 추가:
   ```python
   ("체류", "등록", "F-7", ""): {
       "main": ["통합신청서", "비취업 서약서", "F7_등록"],
       "agent": ["위임장", "대행업무수행확인서"],
   },
   ```
5. 백엔드 재시작 (`Ctrl+C` 후 다시 실행)

### Tesseract 경로 오류 (Windows)

**증상**: `pytesseract.pytesseract.TesseractNotFoundError`

`backend/routers/scan.py` 또는 `pages/page_scan.py`에서:
```python
pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'
```
이 경로가 실제 설치 위치와 일치하는지 확인. 다르면 수정.

## 12. 보안 주의사항

1. **JWT_SECRET_KEY**는 절대 외부에 노출하지 마세요. `.env` 파일을 GitHub에 올리지 마세요.
2. **서비스 계정 키 JSON** 파일도 외부에 노출하지 마세요.
3. Google Sheets를 "링크가 있는 모든 사람에게 공유"로 설정하면 누구나 데이터를 볼 수 있습니다. 반드시 서비스 계정에만 공유하세요.
4. 관리자 계정 비밀번호는 8자 이상, 영문+숫자+특수문자 조합 권장.
5. 운영 서버의 `ALLOWED_ORIGINS`에 불필요한 도메인을 추가하지 마세요.

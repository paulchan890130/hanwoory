## 3. 화면별 역할 요약

### 3-1. 대시보드 (홈)
- **역할**: 사무소 일별/월별 현황 한눈에 파악
- **핵심 요소**:
  - 당일/이달 접수 건수 카드
  - 긴급 알림 (체류 만기 임박 고객)
  - 공지사항 / 업데이트 피드
  - 주요 업무 빠른 입력 버튼 (체류등록, 체류연장, 귀화 등)
- **데이터 출처**: WCF 서비스 조회 (soa.eos21.co.kr)

### 3-2. 업무 미리보기
- **역할**: 특정 업무를 시작하기 전 필요 서류·절차·안내를 미리 확인하는 가이드 엔진
- **핵심 요소**:
  - 3단계 선택 (업무분류 → 업무항목 → 세부코드)
  - 선택 후 자동 채움 요약 (분류명, 업무명, 코드)
  - 업무 개요 텍스트 패널
  - 하단 2분할: 서식 서류 목록 | 첨부 서류 목록
  - SMS 발송 버튼 (해당 업무용 템플릿 연동)
- **연결**: 고객 선택 → 문서 자동작성 → SMS 발송 → 업무 등록

### 3-3. 고객 관리
- **역할**: 외국인 고객 DB 관리
- **핵심 요소**:
  - 등록이름 (한글), 성/이름 (영문)
  - 외국인등록번호, 여권번호
  - 체류자격, 체류만기일
  - 여권 발급일/만기일
  - 연락처, 주소

### 3-4. 서류 작성
- **역할**: 업무 유형 및 고객 선택 → PDF/HWP 자동 생성
- **근거**: signpoints 파일에 ~130개 서식의 서명 좌표 존재
  - 서명/도장 위치를 좌표로 저장 (서식명|페이지|X|Y)
  - iTextSharp로 PDF 생성, HwpCtrl로 HWP 생성

### 3-5. SMS / 알림
- **역할**: 고객에게 업무 진행 상황 또는 안내 문자 발송
- **채널**: SMS, LMS, MMS, 카카오 알림톡, 카카오 친구톡
- **기능**: 템플릿 선택, 고객 자동 연동, 발송 이력 관리

### 3-6. 대장 출력
- **역할**: 법정 서식 대장 Excel 출력
- **서식**:
  - 출입국민원 접수 대장 (7열 구조)
  - 외국인등록증 교부 대장 (6열 구조)
  - 대행업무 수행확인서 내장

## 4. 업무 흐름 (Workflow Summary)

```
고객 접수
    ↓
업무 미리보기 확인 (필요 서류 확인)
    ↓
고객 정보 조회/등록
    ↓
서류 자동작성 (PDF/HWP 생성)
    ↓
업무 등록 (진행업무 추가)
    ↓
출입국관리사무소 제출
    ↓
업무 상태 업데이트 (접수 → 처리 → 완료)
    ↓
SMS 발송 (완료 안내)
    ↓
결산 반영 (일일/월간)
    ↓
대장 출력 (민원접수대장, 교부대장)
```

- 하이코리아(hikorea.go.kr) 자동 접속 추정
- 민원 접수 현황 자동 조회 가능성
- 체류자격 확인 자동화 가능성

⚠️ 이는 드라이버 존재에 기반한 추정이며 실행 없이 확인하지 않음.

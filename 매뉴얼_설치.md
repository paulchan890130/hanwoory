## 1. 전체 구조 설명

### 이 프로그램은 어떻게 생겼나요?

```
[브라우저 화면]          ←→       [Next.js 프론트엔드]
                                         ↕ API 요청
                                  [FastAPI 백엔드]
                                         ↕
                             [Google Sheets (데이터 저장)]
                             [templates/*.pdf (문서 양식)]
```

프로그램은 두 부분으로 나뉩니다.

**프론트엔드 (frontend/)**
- Next.js 14로 만들어진 웹 화면입니다.
- 브라우저에서 `http://localhost:3000` 으로 접속하면 보이는 부분입니다.
- 버튼, 표, 입력창 등 눈에 보이는 모든 것이 여기 있습니다.

**백엔드 (backend/)**
- FastAPI로 만들어진 서버입니다.
- 프론트엔드가 데이터를 달라고 요청하면 Google Sheets에서 읽어서 응답합니다.
- `http://localhost:8000` 에서 실행됩니다.
- 프론트엔드와 백엔드는 서로 같은 컴퓨터에서 실행되어야 합니다.

**Google Sheets (데이터베이스 역할)**
- 고객 데이터, 업무 목록, 메모, 일정이 모두 Google Sheets에 저장됩니다.
- 일반 엑셀처럼 생겼지만, 프로그램이 자동으로 읽고 씁니다.
- 사람이 직접 열어서 볼 수도 있습니다.

**실행 순서**
1. 백엔드 서버 실행 → `http://localhost:8000` 에서 대기
2. 프론트엔드 실행 → `http://localhost:3000` 에서 대기
3. 브라우저에서 `http://localhost:3000` 접속
4. 로그인 → 화면 로드 → 백엔드에 데이터 요청 → Google Sheets에서 읽어서 화면에 표시

### 2-3. Google 서비스 계정 키 준비

프로그램이 Google Sheets에 접근하려면 **서비스 계정 키** 파일이 필요합니다.

#### 서비스 계정 키 파일이란?
- JSON 형식의 파일입니다 (예: `hanwoory-9eaa1a4c54d7.json`)
- 이미 팀에서 발급받아 사용하고 있다면 그 파일을 사용합니다.
- 새로 만들어야 한다면 아래 절차를 따르세요.

#### 새로 만드는 방법
1. https://console.cloud.google.com/ 에 접속 → 로그인
2. 왼쪽 메뉴 → **IAM 및 관리자** → **서비스 계정** 클릭
3. **서비스 계정 만들기** 클릭
4. 이름 입력 (예: `kid-backend`) → 만들기
5. 만든 계정 클릭 → **키** 탭 → **키 추가** → **새 키 만들기** → **JSON** 선택 → 만들기
6. JSON 파일이 자동 다운로드됨

#### Google Sheets API 활성화
1. https://console.cloud.google.com/ → 왼쪽 메뉴 → **API 및 서비스** → **라이브러리**
2. "Google Sheets API" 검색 → **사용** 클릭
3. "Google Drive API" 검색 → **사용** 클릭

#### Sheets 공유 설정
Google Sheets 스프레드시트를 열고 → 우상단 **공유** 버튼 → 서비스 계정의 이메일 주소(JSON 파일 안의 `client_email` 값)를 **편집자** 권한으로 추가.

```
source .venv/bin/activate
```

활성화되면 프롬프트 앞에 `(.venv)` 가 표시됩니다:

## 5. 최초 계정 생성

처음 실행하면 Accounts 시트에 계정이 없습니다.

### 방법 1: 회원가입 (일반 직원용)

1. `http://localhost:3000` 접속
2. 로그인 화면 하단 **"회원가입"** 클릭
3. 아이디, 비밀번호 입력 → 가입 완료
4. 단, 관리자 승인이 있어야 로그인이 가능할 수 있음 (시스템 설정에 따라 다름)

### 방법 2: 관리자 계정 직접 생성 (API 사용)

처음 한 번은 아래 방법으로 관리자 계정을 만듭니다.

브라우저에서 `http://localhost:8000/docs` 접속 → Swagger UI에서:
1. `/api/admin/create-account` 엔드포인트 찾기
2. **Try it out** 클릭
3. Request body에 아래 입력:
```json
{
  "login_id": "admin",
  "password": "your-password",
  "display_name": "관리자",
  "tenant_id": "hanwoory",
  "is_admin": true
}
```
4. **Execute** 클릭
5. 응답 `{"message": "계정 생성 완료"}` 확인

이후 `http://localhost:3000` 에서 `admin` / `your-password` 로 로그인.

## 6. 자주 막히는 문제와 해결법

### 로그인이 안 될 때

**증상**: 아이디/비밀번호 입력 후 "인증 실패" 또는 화면이 바뀌지 않음

확인 순서:
1. 백엔드가 실행 중인지 확인 → `http://localhost:8000/health` 접속해서 `{"status":"ok"}` 나오는지 확인
2. 계정이 Accounts 시트에 있는지 확인 → Google Sheets에서 직접 열어 `Accounts` 탭 확인
3. 비밀번호 해시 방식이 맞는지 확인 → 최근 등록된 계정의 `password_hash` 값이 `base64`로 시작하는지 확인 (80자 이상이면 정상)

**증상**: `CORS` 오류 (브라우저 개발자도구 콘솔에 빨간 글씨)

`.env`에서 `ALLOWED_ORIGINS=http://localhost:3000` 설정 확인. 포트가 3000이 아니면 실제 포트로 바꿈.

---

### Google Sheets 연결 오류

**증상**: `google.auth.exceptions.DefaultCredentialsError`

`config.py`의 `KEY_PATH` 경로 확인. 파일이 실제로 그 경로에 있는지 확인:
```
dir "C:\Users\윤찬\한우리 현행업무\프로그램\출입국업무관리\hanwoory-9eaa1a4c54d7.json"
```
파일이 없으면 "파일을 찾을 수 없습니다" 출력. 경로를 수정하거나 파일을 해당 경로로 이동.

**증상**: `gspread.exceptions.SpreadsheetNotFound`

Sheets가 서비스 계정과 공유되지 않은 것. Sheets 공유 설정에서 `client_email` 값을 편집자로 추가.

### 포트 충돌

**증상**: `Address already in use` 또는 `EADDRINUSE: address already in use :::3000`

이미 해당 포트를 사용하는 프로그램이 있는 것입니다.

**Windows에서 포트 사용 프로세스 종료:**
```
netstat -ano | findstr :8000
taskkill /PID [위에서 나온 PID 번호] /F
```

# 백엔드를 8001로 실행
python -m uvicorn backend.main:app --reload --port 8001

### OCR 결과가 비정상일 때

**증상**: 여권 스캔했는데 이름이 이상하거나 빈칸

`kor` 과 `eng` 가 목록에 있어야 합니다. 없으면 Tesseract 재설치 시 Korean 언어 데이터 체크.

OCR 정확도는 이미지 품질에 따라 다릅니다. 사진이 흐리거나 기울어지면 정확도가 낮아집니다.

[ ] 문서 자동작성 → PDF 다운로드 확인
[ ] 통합검색 → 고객 이름 검색 결과 확인

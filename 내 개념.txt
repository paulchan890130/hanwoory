이렇게 배치되는 개념이야, 이렇게 한 의도는
신 고객 데이터 (구글시트)는 어드민만 사용하고 출입국업무관리 시스템 주요 데이터, 어드민 사무소 고객 데이터, 어드민 사무소 유지 데이터, 텐엔트 계정 데이터를 축적하는 개념이고 신 고객 데이터 (구글시트)는 말그대로 업무하며 그때그때 바뀌는 부분을 수정 정리하고 메모장만큼 수시로 지우지는 않지만 지울 수 있고 자주 참조하며 너느정도 누적데이터도 필요하며 누군가에게 제공할 수도 있는 정제된 자료인 개념이야

따라서 텐엔트 계정 생성 시 신 고객 데이터 (구글시트), 신 업무정리 (구글시트) 의 뼈대가 있어야 시스템을 활용할 수 있지만 내 사우소의 고객정보와 시스템 권한 전체 그리고 업무비밀을 공유할수는 업으니 뼈대가 되는 기본 기준 고객 데이터 (구글시트), 기준 업무정리 (구글시트) 를 두어 이를 기준으로 텐엔트 계정 활성화 시 offices(폴더)에 ID (텐엔트 사무소 폴더) 이 생성되며 그 폴더 안에 ID 고객 데이터 (텐엔트 사무소 구글시트), ID 업무정리 (텐엔트 사무소 구글시트) 를 자동 생성케 하고 API 권한을 주어서 활용할 수 있게한거야
